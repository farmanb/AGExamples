    Section 3 Monomial Orders and Division

    These are some examples of various monomial orderings with Sage, and an implementation of the division algorithm for multivariate polynomial rings (p. 65 of the text).
    🔗

    Subsection 3.1 Monomial Orders

    The point of monomial orders is to export computations from univariate polynomial rings to analogous computations for multivariate polynomial rings. In SageMath, monomial orders are called term orders, and several are implemented in the main libraries. For our purposes, we will primarily be concerned with SageMath’s lex, deglex (graded lex), and degrevlex (graded reverse lex order).
    🔗

    Remark 3.1.

    The default monomoial order in SageMath is degrevlex.
    🔗
    🔗

    Example 3.2.

    The monomial ordering is passed as a named argument to the PolynomialRing constructor and is bound to the polynoial ring. Note that PolynomialRing is immutable once constructed, so you will have to simply re-construct a polynomial ring if you wish to change the term order.

    Subsection 3.2 Multivariate Division Algorithm

    Below is an implementation of the division algorithm. In order to use this function in other cells on the page, you need to click the Evaluate (Sage) button.
    🔗

    Example 3.3.

    Using lex order, divide the polynomial \(f = xy^2 + 1\) by \(f_1 = xy + 1\) and \(f_2 = y + 1\text{.}\)
    🔗
    🔗

    Example 3.4.

    Using lex order, divide the polynomial \(f = x^2 y + xy^2 + y^2\) by the polynomials \(f_1 = xy - 1\) and \(f_2 = y^2 - 1\text{.}\)
    🔗
    🔗

    Example 3.5.

    Using lex order, divide the polynomial \(f = x^2 y + xy^2 + y^2\) by the polynomials \(f_1 = f_1 = y^2 - 1\) and \(f_2 = xy - 1\text{.}\)
    🔗
    🔗

    Remark 3.6.

    Note that Example 4 and Example 5 imply that the ordering of the \(s\)-tuple \(F = (f_1, \ldots, f_s)\) matters when dividing a polynomial \(f\) by \(F\text{.}\)
    🔗
    🔗

    Example 3.7.

    Using lex order, divide the polynomial \(f = x^2 y + xy^2 + y^2\) by the polynomials \(f_1 = y^2 - 1\) and \(f_2 = xy - 1\text{.}\) Then divide \(f\) by \(f_1 = xy - 1\) and \(f_2 = y^2 - 1\)
    🔗
    🔗

    Remark 3.8.

    Note that Example 7 implies the membership problem for multivariate polynomials rings is more subtle than for univariate polynomial rings! In \(k[x]\text{,}\) we saw that every ideal \(I \subseteq k[x]\) is generated by some \(g \in k[x]\text{.}\) Hence for all \(f \in k[x]\text{,}\) \(f \in I \iff g \mid f\text{.}\)
    🔗
    The analogous statement for multivariate polynomials rings is the following. Let \(I = \langle f_1, \ldots, f_s\rangle \subseteq k[x_1, \ldots, x_n]\) be a finitely generated ideal. For all \(f \in k[x_1, \ldots, x_s]\text{,}\) \(f \in I\) if and only if the ordered \(s\)-tuple \(F = (f_1, \ldots, f_s)\) divides \(f\)—i.e., the remainder on division of \(f\) by \(F\) is zero. It is clear that if the remainder on division of \(f\) by \(F\text{,}\) then \(f \in I\text{.}\) However, the converse is false.
    🔗
    Changing the order of the generators does not change the ideal, so \(I = \langle xy - 1, y^2 - 1\rangle = \langle y^2 - 1, xy - 1 \rangle\text{.}\) The second computation above shows us that
    \begin{equation*} f = x(y^2 - 1) + 0(xy - 1) \in I\text{,} \end{equation*}
    even though the reaminder on division by \(F = (xy - 1, y^2 - 1)\) is not zero. This suggests we require more sophisticated machinery to answer the multivariate ideal membership problem.
