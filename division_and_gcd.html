    Section 2 Polynomials of a Single Variable

    Subsection 2.1 Constructing Rings and Fields

    Below are some basic examples of how to use some of the fields and rings available to you in Sage. Most of our computations will happen over \(\Q\text{,}\) but Sage implements all the sets \(\N\text{,}\) \(\Z\text{,}\) \(\Q\text{,}\) \(\R\text{,}\) and \(\C\text{,}\) which are accessed by doubling the letter, as below.
    🔗
    Both RR and CC using 53 bit precision, so working over either field is often quite messy. Unless your polynomial has transcendental coefficients (e.g. \(\pi\text{,}\) \(e\text{,}\) etc.), the algebraic closure of \(\Q\text{,}\) QQbar, will do everything you might want from RR or CC with fewer decimals.
    🔗
    If you know a some Field/Galois theory, Sage is also capable of constructing and working over number fields (i.e. fields \(K\) with \(\Q \subseteq K \subseteq \bar{\Q}\)). As a simple example, we can adjoin the sixth roots of unity—i.e. the complex solutions to \(x^6 - 1\) to the field \(Q\) using the NumberField constructor. Note that you need at least two different variables here. We use x to specify the variable for the defining polynomial and zeta for the primitive sixth root of unity.
    🔗

    Subsection 2.2 Polynomial Rings

    We can form polynomial rings using the construction Name.<var_1, ..., var_n> = Field[], where Field is some field from Sage and Name is a name of your choosing, e.g. \(R\) (for ring).
    🔗
    If for some reason you need to work over a number field, you’ll need to either unbind the variable x or add in a third variable to get polynomials over the number field. For example, we can see that adjoing any primitive sixth root of unity, the polynomial \(x^6 - 1\) splits completely:
